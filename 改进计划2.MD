# ResumePoint 改进计划2 - 深度优化方案（已修订）

## 文档概述

本文档基于六维度深度分析结果，针对ResumePoint iOS应用进行第二轮优化规划。优化重点聚焦于提升美国市场用户体验、增强核心功能竞争力、完善iOS生态系统集成。

> **修订说明**：本文档已根据现有代码库全面审查，修复了以下问题：
> 1. Widget模块中`ProgressRingView`与现有`ProgressRing`重复 → 改为复用`ProgressRing`
> 2. Widget模块中直接创建`NSPersistentContainer` → 改为复用`CoreDataStack` + App Group共享
> 3. Widget模块中缺少`VideoProgress+Widget`扩展和`ConfigurationAppIntent`定义 → 补充完整
> 4. Widget模块中`ProgressBarView`在Widget Extension中不可用 → 需将共享文件加入Widget Target
> 5. Continue Watching模块中`HomeViewModel.markComplete`方法 → 确认`ProgressStoring`协议已有`markCompleted`，补充ViewModel扩展
> 6. Continue Watching模块中`HomeView`集成方式 → 适配现有`Group`结构改为`ScrollView`
> 7. Live Activities模块中`LiveActivityService`未集成到`DependencyContainer` → 补充集成方案
> 8. Live Activities模块中`formatTime`与现有`Double.formattedTime()`重复 → 改为复用
> 9. 统计面板模块中`StatisticsService`未集成到`DependencyContainer` → 补充集成方案
> 10. 统计面板模块中`ContentView`缺少Statistics Tab → 补充Tab集成方案
> 11. Siri模块中`DependencyContainer.shared`不存在 → 改为添加`DependencyContainer.current`静态引用
> 12. Siri模块中`ProgressStoring.addVideo`参数签名不匹配 → 修正为与现有协议一致
> 13. UI/UX模块中`EmptyStateView`修改方案与现有签名不一致 → 适配现有`var`属性
> 14. 测试用例中`MockProgressRepository`方法签名与现有协议不匹配 → 修正为与`ProgressRepositoryProtocol`一致
> 15. Widget使用`IntentTimelineProvider`但iOS 17以下不支持 → 改为`TimelineProvider`兼容方案

## 模块一：Widget小组件系统

### 1.1 需求描述

**用户痛点**：美国iOS用户习惯在主屏幕快速访问常用功能，当前应用缺少Widget支持，用户需要打开App才能查看观看进度。

**解决方案**：提供三种尺寸的Widget，展示最近观看的视频进度，支持一键跳转续看。

**目标用户价值**：
- 减少操作步骤：从主屏幕直接续看
- 提升留存率：Widget常驻增加使用频率
- 符合iOS生态：满足用户对Widget的期望

### 1.2 功能规格

#### Widget类型

| 类型 | 尺寸 | 显示内容 | 交互 |
|------|------|----------|------|
| Small | small | 单个视频进度环 + 标题 | 点击跳转详情 |
| Medium | medium | 3个视频列表 + 进度条 | 点击跳转对应视频 |
| Large | large | 5个视频列表 + 统计数据 | 点击跳转对应视频 |

#### 数据刷新策略

- **Timeline策略**：每15分钟刷新一次
- **后台刷新**：App进入后台时更新Widget数据
- **用户触发**：用户添加/更新视频时立即刷新

#### ⚠️ Widget Extension架构约束

> **重要**：Widget运行在独立Extension进程中，无法直接访问主App的`DependencyContainer`。
> 需要通过**App Group**共享CoreData数据库和UserDefaults。
> Widget Extension中无法直接引用主App的SwiftUI视图组件（如`ProgressRing`、`ProgressBarView`），
> 需要在Widget Target中包含共享源文件（将共享代码文件同时加入主App和Widget两个Target）。

#### 1.4.4 Timeline提供者（复用CoreDataStack + App Group共享）

```swift
// Widget/Models/WidgetTimelineProvider.swift
import WidgetKit
import CoreData

struct VideoProgressTimelineProvider: TimelineProvider {
    typealias Entry = VideoProgressEntry

    func placeholder(in context: Context) -> VideoProgressEntry {
        .placeholder
    }

    func getSnapshot(
        in context: Context,
        completion: @escaping (VideoProgressEntry) -> Void
    ) {
        let entry = VideoProgressEntry(
            date: .now,
            videos: WidgetVideoItem.placeholderData
        )
        completion(entry)
    }

    func getTimeline(
        in context: Context,
        completion: @escaping (Timeline<VideoProgressEntry>) -> Void
    ) {
        Task {
            let videos = await fetchRecentVideos(limit: 5)
            let entry = VideoProgressEntry(
                date: .now,
                videos: videos
            )

            let nextUpdate = Calendar.current.date(
                byAdding: .minute,
                value: 15,
                to: .now
            ) ?? .now

            let timeline = Timeline(
                entries: [entry],
                policy: .after(nextUpdate)
            )
            completion(timeline)
        }
    }

    private func fetchRecentVideos(limit: Int) async -> [WidgetVideoItem] {
        // 复用CoreDataStack创建模型，通过App Group访问共享数据库
        let model = CoreDataStack.createModel()
        let container = NSPersistentContainer(
            name: "ResumePointDataModel",
            managedObjectModel: model
        )

        // 配置App Group共享存储路径
        let storeURL = AppGroupConstants.sharedContainerURL
            .appendingPathComponent("ResumePointDataModel.sqlite")
        let description = NSPersistentStoreDescription(url: storeURL)
        container.persistentStoreDescriptions = [description]

        container.loadPersistentStores { _, error in
            if let error = error {
                print("Widget CoreData failed: \(error.localizedDescription)")
            }
        }

        let request = VideoProgress.fetchRequest() as NSFetchRequest<VideoProgress>
        request.sortDescriptors = [
            NSSortDescriptor(keyPath: \VideoProgress.lastUpdated, ascending: false)
        ]
        request.fetchLimit = limit
        request.predicate = NSPredicate(format: "isCompleted == NO")

        do {
            let results = try container.viewContext.fetch(request)
            return results.map { $0.toWidgetItem() }
        } catch {
            return []
        }
    }
}
```

## 模块二：Continue Watching快捷入口

### 2.1 需求描述

**用户痛点**：美国用户期望流媒体应用首页有"Continue Watching"快捷入口，当前应用缺少此功能，用户需要滚动列表查找最近观看的视频。

**解决方案**：在HomeView顶部添加"Continue Watching"卡片区域，展示最近观看的视频，支持横向滑动浏览。

**目标用户价值**：
- 减少操作步骤：首页即可看到最近观看
- 符合用户习惯：Netflix、Disney+等主流应用都有此功能
- 提升续看率：快捷入口增加续看概率

### 2.2 功能规格

#### 显示规则

| 规则 | 描述 |
|------|------|
| 显示条件 | 存在未完成的视频 |
| 显示数量 | 最近5个未完成视频 |

#### 3.4.2 Live Activity服务

```swift
// Core/Services/LiveActivityService.swift
import ActivityKit
import Foundation

@available(iOS 16.2, *)
protocol LiveActivityManaging {
    func startActivity(for video: VideoProgress) throws -> String
    func updateActivity(id: String, progress: Double, isPlaying: Bool) throws
    func endActivity(id: String) throws
    func endAllActivities() throws
}

@available(iOS 16.2, *)
final class LiveActivityService: LiveActivityManaging {

    private var currentActivity: Activity<WatchingActivityAttributes>?

    func startActivity(for video: VideoProgress) throws -> String {
        guard ActivityAuthorizationInfo().areActivitiesEnabled else {
            throw LiveActivityError.activitiesNotEnabled
        }

        if let existing = currentActivity {
            existing.end(using: nil, dismissalPolicy: .immediate)
        }

        let attributes = WatchingActivityAttributes(videoId: video.id)
        let initialState = WatchingActivityAttributes.ContentState(
            videoTitle: video.title,
            platform: video.streamingPlatform.displayName,
            platformColor: video.streamingPlatform.accentColor,
            currentPosition: video.currentPosition,
            totalDuration: video.totalDuration,
            progressPercentage: video.progressPercentage,
            isPlaying: true,
            lastUpdated: Date()
        )

        let activity = try Activity.request(
            attributes: attributes,
            content: .init(state: initialState, staleDate: nil),
            pushType: nil
        )

        currentActivity = activity
        return activity.id
    }

    func updateActivity(id: String, progress: Double, isPlaying: Bool) throws {
        guard let activity = currentActivity, activity.id == id else {
            throw LiveActivityError.activityNotFound
        }

        let updatedState = WatchingActivityAttributes.ContentState(
            videoTitle: activity.content.state.videoTitle,
            platform: activity.content.state.platform,
            platformColor: activity.content.state.platformColor,
            currentPosition: activity.content.state.currentPosition,
            totalDuration: activity.content.state.totalDuration,
            progressPercentage: progress,
            isPlaying: isPlaying,
            lastUpdated: Date()
        )

        Task {
            await activity.update(using: .init(state: updatedState, staleDate: nil))
        }
    }

    func endActivity(id: String) throws {
        guard let activity = currentActivity, activity.id == id else {
            throw LiveActivityError.activityNotFound
        }

        Task {
            await activity.end(using: nil, dismissalPolicy: .default)
        }
        currentActivity = nil
    }

    func endAllActivities() throws {
        Task {
            for activity in Activity<WatchingActivityAttributes>.activities {
                await activity.end(using: nil, dismissalPolicy: .immediate)
            }
        }
        currentActivity = nil
    }
}

enum LiveActivityError: LocalizedError {
    case activitiesNotEnabled
    case activityNotFound

    var errorDescription: String? {
        switch self {
        case .activitiesNotEnabled:
            return "Live Activities are not enabled on this device"
        case .activityNotFound:
            return "No active Live Activity found with the specified ID"
        }
    }
}
```

#### 4.4.2 统计服务（复用ProgressRepositoryProtocol）

```swift
// Core/Services/StatisticsService.swift
import Foundation
import CoreData

protocol StatisticsCalculating {
    func calculateStatistics() async throws -> WatchingStatistics
    func calculatePlatformDistribution() async throws -> [WatchingStatistics.PlatformStats]
    func calculateWeeklyTrend() async throws -> [WatchingStatistics.DailyStats]
}

final class StatisticsService: StatisticsCalculating {

    private let repository: ProgressRepositoryProtocol

    init(repository: ProgressRepositoryProtocol) {
        self.repository = repository
    }

    func calculateStatistics() async throws -> WatchingStatistics {
        let allVideos = try await repository.fetchAll()

        let totalWatchTime = allVideos.reduce(0) { $0 + $1.currentPosition }
        let completedVideos = allVideos.filter { $0.isCompleted }
        let completionRate = allVideos.isEmpty ? 0 : Double(completedVideos.count) / Double(allVideos.count) * 100
        let averageProgress = allVideos.isEmpty ? 0 : allVideos.reduce(0) { $0 + $1.progressPercentage } / Double(allVideos.count)

        let weekAgo = Calendar.current.date(byAdding: .day, value: -7, to: Date()) ?? Date()
        let weeklyVideos = allVideos.filter { $0.lastUpdated >= weekAgo }
        let weeklyWatchTime = weeklyVideos.reduce(0) { $0 + $1.currentPosition }

        let platformDistribution = try await calculatePlatformDistribution()
        let weeklyTrend = try await calculateWeeklyTrend()

        return WatchingStatistics(
            totalWatchTime: totalWatchTime,
            weeklyWatchTime: weeklyWatchTime,
            completionRate: completionRate,
            averageProgress: averageProgress,
            totalVideos: allVideos.count,
            completedVideos: completedVideos.count,
            platformDistribution: platformDistribution,
            weeklyTrend: weeklyTrend
        )
    }

    func calculatePlatformDistribution() async throws -> [WatchingStatistics.PlatformStats] {
        let allVideos = try await repository.fetchAll()

        let grouped = Dictionary(grouping: allVideos) { $0.platform }
        return grouped.map { platform, videos in
            let streamingPlatform = StreamingPlatform(rawValue: platform) ?? .other
            return WatchingStatistics.PlatformStats(
                platform: streamingPlatform.displayName,
                count: videos.count,
                watchTime: videos.reduce(0) { $0 + $1.currentPosition },
                color: streamingPlatform.accentColor
            )
        }.sorted { $0.count > $1.count }
    }

    func calculateWeeklyTrend() async throws -> [WatchingStatistics.DailyStats] {
        let calendar = Calendar.current
        let today = calendar.startOfDay(for: Date())

        var trend: [WatchingStatistics.DailyStats] = []

        for i in 0..<7 {
            guard let date = calendar.date(byAdding: .day, value: -i, to: today) else { continue }
            let nextDate = calendar.date(byAdding: .day, value: 1, to: date) ?? date

            let predicate = NSPredicate(
                format: "lastUpdated >= %@ AND lastUpdated < %@",
                date as NSDate,
                nextDate as NSDate
            )
            let dayVideos = try await repository.fetch(by: predicate, sortDescriptors: nil)

            let stats = WatchingStatistics.DailyStats(
                date: date,
                watchTime: dayVideos.reduce(0) { $0 + $1.currentPosition },
                videosWatched: dayVideos.count
            )
            trend.append(stats)
        }

        return trend.reversed()
    }
}
```

#### 4.4.3 StatisticsViewModel（复用Double.formattedTime()）

```swift
// Features/Statistics/StatisticsViewModel.swift
import SwiftUI

@MainActor
final class StatisticsViewModel: ObservableObject {
    @Published var statistics: WatchingStatistics = .empty
    @Published var isLoading = false

    private let statisticsService: StatisticsCalculating

    init(statisticsService: StatisticsCalculating) {
        self.statisticsService = statisticsService
    }

    var formattedTotalTime: String {
        statistics.totalWatchTime.formattedTime() // 复用现有Double.formattedTime()
    }

    var formattedWeeklyTime: String {
        statistics.weeklyWatchTime.formattedTime() // 复用现有Double.formattedTime()
    }

    func loadStatistics() async {
        isLoading = true
        do {
            statistics = try await statisticsService.calculateStatistics()
        } catch {
            statistics = .empty
        }
        isLoading = false
    }
}
```

## 模块五：Siri快捷指令集成

### 5.1 需求描述

**用户痛点**：用户希望通过语音或快捷指令快速操作应用，当前应用缺少Siri集成。

**解决方案**：集成App Intents，支持语音命令和快捷指令。

**目标用户价值**：
- 语音操作：通过Siri控制应用
- 快捷指令：iOS快捷指令App集成
- 自动化：支持自动化工作流

### 5.2 功能规格

#### 支持的意图

| 意图 | 触发词 | 功能 |
|------|--------|------|
| ContinueWatching | "继续观看我的节目" | 打开最近观看的视频 |
| AddVideo | "添加视频到ResumePoint" | 添加新视频 |
| ShowStatistics | "显示我的观看统计" | 打开统计页面 |
| MarkComplete | "标记当前视频完成" | 标记最近视频完成 |

#### ⚠️ App Intents数据访问约束

> **重要**：App Intents运行在主App进程中，可以直接访问主App代码。
> 但当前`DependencyContainer`通过`@StateObject`在`ResumePointApp`中创建，没有全局访问点。
> 解决方案：在`DependencyContainer`中添加`static var current`静态引用，在`init`时设置。
